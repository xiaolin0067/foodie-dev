因为图的存储表达方式太多了，所以遇到不会的表达方式的时候，将其转换为自己会的表达方式来解题

图的宽度(广度)优先遍历-BFS
  1，利用队列实现
  2，从源节点开始依次按照宽度进队列，然后弹出
  3，每弹出一个点，把该节点所有没有进过队列的邻接点放入队列
  4，直到队列变空
深度优先遍历-DFS
  1，利用栈实现
  2，从源节点开始把节点按照深度放入栈，然后弹出
  3，每弹出一个点，把该节点下一个没有进过栈的邻接点放入栈
  4，直到栈变空

拓扑排序算法
  项目A依赖项目B，项目B依赖项目C，各个项目之前不能循环依赖，将需要先编译的项目排序到前面
  适用范围: 要求有向图，且有入度为0的节点，且没有环
  解法：找到入度为0的节点
       处理该节点
       将节点该节点与其影响的边去除
       重新找到入度为0的节点（循环）

生成最小生成树
  最小生成树：保证图所有节点的连通性的前提下，使得所有边的权重之和最小（去除一些边，保证连通性，权重最低）
  kruskal算法（克鲁斯卡尔算法）
    去除图中所有节点的边，将图中所有边排序，从最小的边开始在节点上加上，若加上形成了环则不加
    一开始将图中所有节点拆分为单个节点在一个集合，添加一条边时，判断这条边的from和to节点是否在同一个集合中
    若不在则添加这表边并将这两个节点加在同一个集合中，若在同一个集合则取消添加这条边
    适用范围: 要求无向图
  prim算法（普里姆算法）
    适用范围:要求无向图

Dijkstra算法（狄克斯特拉算法）
  解决有权图中最短路径问题
  适用范围：没有累加和为负数的环，将导致一直转圈来缩小权值

森林：各个图互相不连通形成森林

*并查集结构*：做合并做查询都是常数级别
